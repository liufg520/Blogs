二.Hexo实施步骤
1.下载安装Termux并注册GitHub
Termux下载地址：https://www.coolapk.com/apk/com.termux
GitHub注册地址：https://github.com/
2.更换国内源为清华源
sed -i ‘s@^(deb.stable main)$@#\1\ndeb https://mirrors.tuna.tsinghua.edu.cn/termux/termux-packages-24 stable main@’ $PREFIX/etc/apt/sources.list
sed -i ‘s@^(deb.games stable)$@#\1\ndeb https://mirrors.tuna.tsinghua.edu.cn/termux/game-packages-24 games stable@’ $PREFIX/etc/apt/sources.list.d/game.list
sed -i ‘s@^(deb.*science stable)$@#\1\ndeb https://mirrors.tuna.tsinghua.edu.cn/termux/science-packages-24 science stable@’ $PREFIX/etc/apt/sources.list.d/science.list
apt update && apt upgrade
3.安装常用基础工具
pkg install vim curl wget git tree openssh -y
4.下载安装Nodejs
pkg install nodejs-lts 安装稳定版本
node -v 查看node版本 npm -v 查看npm版本
5.安装并初始化Hexo
npm install hexo-cli -g 使用npm安装hexo
hexo -v 查看版本
PS：安装完成后你会看到有WARN，不用担心，不影响使用。
mkdir hexo 创建一个目录
cd hexo 进入hexo目录
hexo init 将hexo初始化到hexo目录下
6.生成静态文件并启动Hexo
hexo g 生成静态网页存放于public目录下
hexo s 启动hexo 默认端口4000
7.可以修改Hexo默认端口号
vi ~/hexo/node_modules/hexo-server/index.js
找到port字段修改默认端口为8081，如下：
hexo.config.server = Object.assign({ port: 8081,
PS：这里我修改过默认端口所以访问http://localhost:8081，如果能够正常访问，则说明 Hexo 本地博客已经搭建起来了，将hexo关联到github就可以在外网访问您的博客了。
8.配置next主题、语言、菜单、侧栏等特性
参考官网：http://theme-next.iissnan.com/
9.配置Hexo部署到GitHubPages
1)注册GitHub 注意用户密码不要包含@字符，后面会有说明
2)登录GitHub创建仓库 名字必须为：<Github账号名称>.github.io 格式如下：
我之前创建过，所以提示这个名称已存在。如下图：
content://com.bxs.note.fileprovider/external_files/%E9%85%B7%E8%AE%B0%E4%BA%8B/Cache/.img/m_1593691808553.kjs
3)安装hexo-deployer-git插件
npm install hexo-deployer-git –save
注意：在你初始化hexo目录下执行此命令安装插件
4)创建ssh key，密码为空
ssh-keygen -t rsa -C “邮箱地址”
注意：输入以上命令直接回车3下密码为空，邮箱地址是注册登录github使用的邮箱地址，改成你自己的邮箱即可。
5)把生成的公钥添加到GitHub
cat .ssh/id_rsa.pub 查看复制生成的公钥文件内容
6)在GitHub设置页面添加ssh key粘贴公钥内容
content://com.bxs.note.fileprovider/external_files/%E9%85%B7%E8%AE%B0%E4%BA%8B/Cache/.img/m_1593696754827.kjs
7)测试ssh key是否添加成功
ssh -T git@github.com
content://com.bxs.note.fileprovider/external_files/%E9%85%B7%E8%AE%B0%E4%BA%8B/Cache/.img/m_1593696953571.kjs
8)配置 Deployment参数
vi ~/hexo/_config.yml 这是站点配置文件
注意：https后面跟的是github的用户名和密码截止到@符号之前,所以密码中不要包含@符号 否则会报错。

9)配置个人信息告诉github你是谁来进行权限处理
git config –global user.email “9@qq.com“
git config –global user.name “liufg520”
10)测试Hexo是否推送到GithubPages
hexo clean 清除旧public目录文件
hexo g 生成新public目录文件
hexo d 部署推送到github
content://com.bxs.note.fileprovider/external_files/%E9%85%B7%E8%AE%B0%E4%BA%8B/Cache/.img/m_1593699554688.kjs
如果返回以上信息表示部署推送成功，使用浏览器访问https://liufg520.github.io就OK了！！！

1)申请个人域名 PS：可免费使用一年
http://www.freenom.com/zh/index.html
2.在域名管理配置域名解析添加CNAME类型
content://com.bxs.note.fileprovider/external_files/%E9%85%B7%E8%AE%B0%E4%BA%8B/Cache/.img/m_1593701001499.kjs
PS：此时你可以使用你的个人域名来访问你的博客了，是不是很开心呢！
